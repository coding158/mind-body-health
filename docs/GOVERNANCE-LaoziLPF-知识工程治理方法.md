# LaoziLPF 知识工程治理方法 · Governance for Classical-Text Knowledge Engineering
# 一个古籍解释体系知识库如何做到可审计、可复现、多 AI 协作不漂移

> 来自 LaoziLPF 源项目 22 条架构决策记录（ADR）的方法论提炼公开版。
> 源项目:以马王堆帛书《老子》+ 单一解释体系,做逐字级知识工程,截至 2026-07 已完成道经 48 章全部 + 德经 30/68 章,由 Claude / DeepSeek 等多个 AI 模型接力生产、互相审查。
> 本文档不含源项目版权引文,提炼的是**任何"古籍+解释体系"知识库项目都可搬用的治理骨架**。整理：2026-07-21。

---

## 0. 三个立项判断（决定整个项目形态）

1. **知识工程 ≠ 学术注疏**。目标不是"评价解释对不对",而是把一个自成一体的解释体系完整、忠实、可检索地结构化。因此**只选一个解释体系做单一事实源（SSoT）**——混合众家会让每个字都要仲裁,SSoT 崩溃。
2. **价值在知识,不在 Schema**。文件结构/字段定型后冻结,每改一次 Schema = 全库断链 + 后接手的 AI 以为"还该继续改"。
3. **治理强度匹配项目类型**。内容/知识库项目用"轻治理":人工铁律 + 单出口 CI + 定期回审,不装重型流程骨架（护栏永远留,骨架按需启动）。

## 1. 分层冻结（这套方法的地基）

```
L1 图版(影印件)          终审依据,人眼核字
L2 解释者原书            实录来源,照录不转述
L3 释文电子底本           machine-checkable 冻结源:SHA256 + 字符区间(boundary)
────────── 以上冻结,任何工具不得要求修改 ──────────
章节知识文件              原文直录(镜像冻结源) + 逐字注释 + 逐句译文 + 证据表
知识对象注册表(EO)        每个确认训释一个条目:出处/引文/置信度/关系
```

冻结的意义:**下游一切推导都能逐字回溯到不可变的锚点**,AI 产出再多也不会"越写越漂移"。

## 2. 七条生产铁律（多 AI 协作实测有效）

1. **冻结三不动**:冻结源正文、boundary 边界文件、镜像描述头,绝不修改;疑有误→停下报人工。
2. **实录优先 + 照录铁律**:先全文检索解释者全部著作（不是只查目录!）;有原文绝不转述——转述会把自己的推演悄悄混进"实录"标签。
3. **分层标注**:实录/推演逐条标明 + 信心分级;实录与推演冲突→撤推演、留订正痕迹、永不删除。
4. **完成标记门**:完成标志当且仅当全部工序做完且 CI 全绿才翻转,**绝不批量刷**——这是实测拦住事故最多的一条（曾有协作模型批量把未完成章刷成"完成",靠这道门在回审时抓回）。
5. **慎用批量脚本**:批量正则改正文是历史事故高发区;临时脚本一律放仓库外。
6. **独立证据纪律**:字形确认 ≠ 结论确认;没有独立证据的候选永不计数（防自我循环证成）。
7. **层间红线**:外部诠释体系的资料不进正文、不作证据（见实践对照规则)。

## 3. 单出口 CI（"verify-all"模式）

所有机器校验收敛为**一个命令、一个结论**（全绿才许合并）,包括:冻结源完整性（SHA 逐位比对）、章节结构审计、跨章一致性（含"字典绕行"检测——译文提到关键概念却只挂编号不写含义,判 FAIL）、证据链审计、知识对象治理、知识健康分。
**单一出口 = 单一真相**:防止"跑了其中两项就以为全绿"。同时明确**CI PASS ≠ 结论正确**——自动化只查路径完整性,内容正误永远由人工/回审判断。

## 4. 多 AI 协作分工（实测可行的形态）

```
建样模型(能力强)   做黄金样章,定格式与判例
量产模型(成本低)   照样章批量生产
回审模型(能力强)   每 ~10 章抽查:冻结块、来源真伪(繁简两形都搜)、
                  实录/推演标签、完成标记与实际工序是否一致
```

关键经验:**交接文档比模型能力更决定质量**——把铁律、工作流、白名单判例、历史事故写成一份自足的交接文档,任何模型（包括未来的新模型）读完即可接手;每次事故的教训回写进交接文档与 ADR,形成进化闭环。

## 5. ADR:让"为什么"活下来

每个关键决策一条 ADR（为什么选这个解释体系/为什么冻结/为什么这条红线…),新协作者先读 ADR 目录再干活。**ADR 的价值不在记录,在拦截**:后来者(人或 AI)想"优化"架构时,先撞见当年不这么做的理由。

配套节奏:**提出规则 → 人工验证 → CI 自动化 → ADR 冻结 → 进入生产**。闭环即冻结,不"一直优化";动框架的唯一标准是触发条件出现,而不是"还能更好"。
