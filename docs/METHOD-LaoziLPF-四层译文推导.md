## 0. 一句话

古典修炼文本的每一句译文,不该是"文言→白话"的一步黑箱跳跃,而应是**四层输出**——每层独立可核查,读者能看见"这句白话是怎么推出来的":

```
Layer 1  Literal    原文       只依据校勘定本,不越过它揣测异文
Layer 2  Semantic   语义恢复    只依据所选解释体系的逐字训释(含《说文》段注等字书锚点)
Layer 3  Functional 功能推导    严格由 Layer 1+2 推导,不引入外部体系
Layer 4  Practice   实践对照    可参考其他传统(禅、静心、辟谷、现代医学…),但明确标注
                              "实践对照"而非"原文含义",且永不倒灌回 Layer 1-3
```

Layer 1-3 构成**证据链**（逐句强制）；Layer 4 是**独立对照层**（可选,不进入证据链）。

## 1. 四步微推导（Layer 2 → Layer 3 内部怎么走）

以《德经》首句"上德不德,是以有德"为例（字义训释采干昌新体系,以下为**本方法示意的转述**,非原书文句照录）:

```
Step 1  词义恢复
    上 → 人体巅顶(含大脑)          〔解释体系明训〕
    德 → 生命功能的哺育升登         〔《说文注》"升当作登"字源锚〕
    不 → 语助词,无实义             〔解释体系明训,非否定〕
    德 → 同上

Step 2  功能关系
    上德 → 巅顶生命功能进入升登状态

Step 3  逻辑关系
    不德 → 语助足句:全句是判断句"上德,德也",不是"上德却不要德"

Step 4  现代语言
    巅顶大脑的生命功能升登,是靠哺育先天潜能而实现的——这就是"有德"。
    （转述;源项目内此句有解释者本人的整句译文,遵"照录优先"原则直接采用原文）
```

四步的关键纪律:

1. **先功能后逻辑**——先把每个字的功能义组合成短语,再处理语助/转折/判断等句法,不许跳步直接造白话。
2. **白话是转写不是翻译**——Step 4 的每个词都必须能回溯到 Step 1 的字义,不得为了"通顺"引入前面没出现过的新概念。
3. **有整句原译先照录**——解释体系原书里有现成整句译文的,直接引用并标实录,不重新推导（"有答案不猜答案"）;无原译才走四步推演,并标明"推演"与信心等级。
4. **同字异位不同训**——同一个字在不同位置允许不同读法,但每次都要写明判据,不许悄悄换。

## 2. 为什么要四层（这套方法解决什么问题）

| 常见问题 | 四层方法的对治 |
|---|---|
| 译文与原文之间是黑箱,读者无法核查 | 每层留痕,推导透明可审计 |
| 不同译者/不同 AI 风格漂移 | 统一推导算法,多协作者产出一致 |
| "实录"与"推演"混在一起,以译代注 | 逐句标注来源与信心等级,冲突时撤推演采实录 |
| 借其他体系的话头"帮助理解",结果混淆了原意 | 外部体系只进 Layer 4 且强制标注,永不倒灌 |
