# 粤语诵读层 · Cantonese Recitation Layer
# 索引与说明 · Index & Notes

> 粤语保留较多中古汉语音韵,诵读古代经典更近其古音、节奏与韵味。
> 本层收录**作者本人用粤语诵读公版经典**的录音,作为文字之外的另一条入口——
> 用耳朵、用声音去亲近经典,而非只用眼睛读。
>
> Cantonese preserves much of Middle-Chinese phonology; reciting the classics in it comes closer to their ancient sound and rhythm. This layer holds the author's own Cantonese recitations of public-domain classics — a way in through the ear and the voice, not only the eye.

---

## 为什么做粤语层 · Why Cantonese

- **音韵更近古**:许多经典的押韵、平仄,在普通话里已散失,粤语里仍可听见。
- **诵读即修行**:出声诵读时,注意力自然收摄于声音与气息,本身就是一种"转触"——
  从念头的空转,回到当下的听与念(参见 [`docs/AWARENESS-觉知与六触.md`])。
- **声音是另一把钥匙**:有人从文字入,有人从声音入;多一条入口,多一个有缘人能进来。

## 录音规格建议 · Recording Specs（自用备忘）

```
分发格式：MP3，128 kbps，单声道（诵读人声足够，文件小、兼容最好）
母带保存：WAV 无损，自己留底，不上传分发
环境：安静、无回声；手机录音 App 即可起步，不必追求专业设备
心态：不求"播音腔"，求清晰、平稳、心在当下——诵读是修行，不是表演
```

## 托管与链接方式 · Hosting & Linking

> 音频文件**不直接塞进 git 仓库**(git 不适合大二进制文件,会拖慢 clone)。
> 推荐:托管在外部,Markdown 里放链接。

可选托管:
- **archive.org(互联网档案馆)** —— 公益、永久、自带播放器;诵读公版经典放这里语境最契合。
- **GitHub Releases** —— 作为 Release 附件上传,不计入 git 历史,适合文件不多时。
- **自有 VPS / 对象存储** —— 你已有的轻量 VPS 也可。

GitHub Markdown 不渲染 `<audio>` 播放器,故用清晰链接:
```markdown
🎧 [点击收听（MP3）](音频链接)
```
